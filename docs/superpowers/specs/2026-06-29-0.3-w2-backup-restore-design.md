# RDDA 0.3 — W2: `rdda backup` / `rdda restore` (encrypted EU state) (design)

## 0.3 roadmap context

0.3 is the operator quality-of-life release (workstreams W1–W5; see
`docs/superpowers/specs/2026-06-28-0.3-w1-health-status-design.md` for the roadmap).
**W1 (health beat + `rdda status`) is shipped.** This is **W2**, fully independent of the
others: disaster-recovery / host-migration for the EU node's state.

## Goal

Let the operator capture the EU node's source-of-truth state into a single **encrypted**
archive and restore it onto a node, so a lost/replaced EU box can be rebuilt and clients keep
working without re-issuing configs.

## Scope

The EU node is the single source of truth. Back up exactly:

- `config.yaml` — node + transport settings, REALITY keypairs, tunnel UUID, **pull token**.
- `clients/` — one file per client (UUID, shortID, name, metadata).

Explicitly **excluded** (derived or transient, never source of truth): `singbox.json`
(rendered), `ru-health.json` (W1 transient beat), `geoip-ru.srs` (RU-only, installer-fetched),
anything under `/etc/cloudflared` (cloudflared's own credentials, outside `/etc/rdda`).

The archive contains **secret material**, so it is encrypted with an operator passphrase.

## Crypto

Dependency-light: argon2id + XChaCha20-Poly1305 are both already available via the pinned
`golang.org/x/crypto`. No new crypto module.

- **Plaintext:** `tar(config.yaml, clients/…)` then gzip. State is KB-sized, so the whole
  gzipped tar is encrypted as **one** AEAD message (no streaming).
- **Key derivation:** `argon2.IDKey(passphrase, salt, time=2, memory=64*1024, threads=4,
  keyLen=32)` with a 16-byte crypto/rand salt.
- **Encryption:** `chacha20poly1305.NewX(key)` (XChaCha20-Poly1305), 24-byte crypto/rand
  nonce, `Seal(nil, nonce, plaintext, header)` — the **header is the associated data** so any
  tampering with magic/version/params/salt/nonce is detected on open.
- **File format** (single file, mode `0600`):

  ```
  magic    "RDDABAK1"      8 bytes
  version  0x01            1 byte
  time     uint32 BE       argon2 time cost
  memory   uint32 BE       argon2 memory (KiB)
  threads  uint8           argon2 parallelism
  salt     16 bytes
  nonce    24 bytes
  ciphertext + 16-byte Poly1305 tag   (rest of file)
  ```

  Argon2 params are stored (not hard-assumed) so a future cost bump can still open old
  archives. The fixed-size header (magic..nonce = 8+1+4+4+1+16+24 = 58 bytes) is read first,
  then used as AEAD AAD.

## Commands (operator-run on the EU node)

### `rdda backup [--out <path>] [--passphrase-file <p>]`
1. Acquire passphrase (precedence): `$RDDA_BACKUP_PASSPHRASE` → `--passphrase-file` (file
   contents, trailing newline trimmed) → **no-echo TTY prompt entered twice** (must match;
   reject empty).
2. `backup.Create(stateDir, passphrase)` → archive bytes.
3. Write to `--out` (default `rdda-backup-<UTC-YYYYMMDDTHHMMSSZ>.rdda` in CWD; `--out -`
   writes the raw archive to stdout). File mode `0600`.
4. Print the output path + a one-line "encrypted; keep your passphrase — it cannot be
   recovered" note to stderr.

`stateDir` is the global `--dir` (default `/etc/rdda`).

### `rdda restore <file> [--force] [--passphrase-file <p>]`
1. Acquire passphrase (env → file → no-echo prompt **once**).
2. Read the archive; `backup.Restore(archive, passphrase, stateDir, force)`:
   - Decrypt + verify (wrong passphrase or tampered/truncated input → error, **nothing
     written**).
   - If `stateDir/config.yaml` exists and `!force` → refuse with a clear message (don't
     clobber a live node).
   - Unpack the gzipped tar into a temp dir under `stateDir`'s parent, then move
     `config.yaml` + `clients/` into place; on any failure the existing state is untouched.
   - `chown rdda:rdda` and `0600`/`0700` the restored files/dir so the EU services can read
     them (best-effort: if the `rdda` user doesn't exist, log a warning and continue).
3. Print what was restored (config + N clients).

## Components (small, isolated, testable)

- **`internal/backup/backup.go`** — `func Create(stateDir, passphrase string) ([]byte, error)`
  and `func Restore(archive []byte, passphrase, destDir string, force bool) error`, plus
  unexported helpers: tar/gzip pack+unpack, `deriveKey`, header marshal/parse, seal/open. No
  CLI, no I/O beyond reading `stateDir` (Create) / writing `destDir` (Restore). Pure logic →
  fully unit-testable.
- **`internal/cli/backup.go`** — `rdda backup` command (passphrase acquisition + `--out`).
- **`internal/cli/restore.go`** — `rdda restore` command (passphrase acquisition + `--force`
  + post-restore chown).
- Both registered in `newRoot()` (`cli.go`). Passphrase acquisition (env/file/prompt) is a
  shared helper in the cli package (e.g. `readPassphrase(confirm bool) (string, error)` using
  `golang.org/x/term` for the no-echo prompt).

## Path traversal safety

When unpacking the tar, reject any entry whose cleaned path escapes the destination (`..`,
absolute paths) — only `config.yaml` and `clients/<name>` entries are allowed. A malicious
archive must not write outside `destDir`.

## Error handling

- Missing/empty passphrase → clear error, no output written.
- Wrong passphrase / tampered / truncated archive → AEAD-open error; restore makes **no**
  changes to the destination.
- `backup`: unreadable `stateDir`/`config.yaml` → error before any output.
- `restore` overwrite without `--force` → refuse; with `--force` → temp-unpack then atomic
  move so a mid-restore failure can't leave a half-written state.

## Dependencies

- **New:** `golang.org/x/term` (no-echo passphrase prompt) — small standard x/ module, added
  with an explicit pinned version (NOT via bare `go mod tidy`).
- argon2id, chacha20poly1305: already in the pinned `golang.org/x/crypto`.

## Testing

**Unit (`internal/backup`):**
- Round-trip: seed a temp `stateDir` with a `config.yaml` + 2 client files → `Create` →
  `Restore` into a fresh dir → restored bytes byte-equal the originals (config + every client).
- Wrong passphrase → `Restore` errors and writes nothing.
- Tampering: flip a ciphertext byte / truncate the file → `Restore` errors (AEAD detects it).
- Header-as-AAD: flip a salt/nonce byte → open fails.
- Overwrite guard: `Restore` into a dir that already has `config.yaml` errors unless `force`.
- Path traversal: a crafted archive entry `../evil` is rejected.

**CLI smoke (`internal/cli`):** `rdda backup --out <tmp> --passphrase-file <p>` then
`rdda restore <tmp> --dir <fresh> --passphrase-file <p>` reproduces the state; wrong
passphrase file → non-zero exit.

No multi-host integration harness change (EU-local, no network).

## Out of scope (later / other WS)

Automated/scheduled backups, off-box upload (S3/rsync), key files instead of passphrases,
backing up RU-side state (RU holds no source of truth). These can layer on `internal/backup`
later without changing its interface.
