# RDDA 0.3 — W3: `rdda doctor` (active diagnostics) (design)

## 0.3 roadmap context

0.3 is the operator quality-of-life release (W1–W5). **W1 (health beat + `rdda status`)** and
**W2 (encrypted `backup`/`restore`)** are shipped. This is **W3**: on-demand *active* diagnostics
that pinpoint where the 2-hop chain is broken. It builds on W1 (reads the stored RU beat) and on
the existing `check-dest` (REALITY-dest reachability).

## Goal

`rdda status` (W1) passively reads local state. `rdda doctor` runs **active probes** —
role-appropriate — and prints a checklist that tells the operator *which link is down* when a
client can't connect, exiting non-zero if anything is broken so it's usable from monitoring/cron.

## Check model

- `type Status` ∈ {`PASS`, `WARN`, `FAIL`}.
- `type Check struct { Name string; Status Status; Detail string; Hint string }` — `Hint` is a
  one-line remediation shown only for non-PASS.
- **Exit rule:** `rdda doctor` prints every check as `✓`(PASS)/`⚠`(WARN)/`✗`(FAIL) with detail
  (+ hint on non-PASS), then **returns an error (non-zero exit) iff any check is FAIL**. WARN
  prints but does **not** fail the exit — WARN means "couldn't verify / degraded", FAIL means
  "provably broken".
- **Role** is auto-detected like `status`: `<dir>/config.yaml` present ⇒ **EU**, else **RU**.

## Role-specific inputs

The two roles hold different state, so doctor reads different sources (it does **not** assume
both exist):

- **EU:** `config.yaml` (via `state.LoadConfig`) — sub hostname, pull token, REALITY/tunnel params.
- **RU:** `singbox.json` (the rendered RU config: inbound `listen_port`, `reality` private_key /
  short_id / server_name / handshake dest, `users[].uuid`) **+** `pull.env`
  (`RDDA_PULL_FROM`, `RDDA_PULL_TOKEN`). RU never has `config.yaml`.

## Architecture

`internal/doctor` is a small package with **injectable seams** so the orchestration, rendering,
and exit logic are unit-testable while the live probes run for real:

```
type Doctor struct {
    dir string
    // seams (real implementations by default; tests override):
    unitActive func(unit string) bool                                   // systemctl is-active
    httpProbe  func(url string) (code int, notAfter time.Time, err error) // GET + peer-cert expiry
    dialDest   func(host string, port int) error                        // TLS 1.3 reachability
    egress     func(singboxConfig []byte, probeURL string) (ok bool, setupErr error) // RU e2e
    cfInfo     func(tunnelID string) (connectors int, err error)        // cloudflared tunnel info
    now        func() time.Time
}
func New(dir string) *Doctor
func (d *Doctor) Run(probeURL string) []Check   // detects role, runs the right check list
```

Pure helpers (unit-tested directly): `Render([]Check) string`, `AnyFail([]Check) bool`.

`internal/cli/doctor.go` builds a `Doctor`, calls `Run`, prints `Render`, and returns a non-nil
error when `AnyFail` (so the process exits non-zero). A `--probe-url` flag overrides the egress
probe target (default a robust public endpoint; the harness points it at the in-network target).

> **Shared logic note:** the TLS-1.3 dial and the "extract REALITY handshake dest from a sing-box
> config" logic also exist (unexported) in `internal/cli/checkdest.go`. `doctor` re-implements
> these two small helpers locally to avoid a `cli`→`doctor` or `doctor`→`cli` dependency; the
> duplication is a few lines each. (A future refactor could lift them into a shared `internal`
> helper; out of scope for W3.)

## Checks

### Core (always run)

**RU:**
1. **units active** — `rdda-singbox` (FAIL if not active); `rdda-nfqws` reported as PASS/INFO if
   present, never FAIL (it's optional/fail-open).
2. **REALITY dest reachable** — `dialDest(host, port)` to the inbound's handshake dest (from
   `singbox.json`). FAIL if unreachable (the inspected hop can't relay), with the same hint
   `check-dest` gives.
3. **control channel up** — `httpProbe(RDDA_PULL_FROM + token)` → 200 expected. FAIL on non-200
   / unreachable (RU can't reach EU's sub endpoint over CF).
4. **last pull fresh** — `singbox.json` mtime; WARN if older than a threshold (e.g. 30 min, ~6×
   the pull interval), else PASS.

**EU:**
1. **units active** — `rdda-singbox`, `rdda-sub`, `cloudflared` (FAIL if any not active).
2. **config renders** — `render ru` and `render eu` succeed (catches a corrupt `config.yaml`).
   FAIL on render error.
3. **public sub endpoint reachable** — `httpProbe("https://<cf-sub-host>/ru/config?token=…")`
   → 200 (the path RU/clients actually hit, end-to-end through CF). FAIL on non-200/unreachable.
   This probe also yields the served cert's expiry (deep probe 3).
4. **RU beat fresh** — read the stored beat (W1 `state.LoadRUHealth`); **WARN** if older than
   20 min (matches W1 staleness) or "no beat yet"; PASS otherwise. (WARN, not FAIL: a stale beat
   strongly suggests the RU node is down but isn't proof from the EU side — the EU-side checks
   above test what EU can prove.)

### Deep probe 1 — RU end-to-end egress (RU, FAIL-capable)

Prove the **whole data path** from inside RU. Build a throwaway **client** config from
`singbox.json`: `server 127.0.0.1`, `server_port` = the inbound `listen_port`, `uuid` =
`users[0].uuid`, REALITY `server_name`/`short_id` from the inbound, and the REALITY **public key
derived from the inbound's private key** via the new `keys.PublicFromPrivate`. Run a real
`sing-box` on an ephemeral loopback SOCKS port, `curl`/GET `probeURL` through it with a timeout,
then kill it.

- **PASS** — the fetch succeeded ⇒ client→RU→EU→internet works end to end.
- **FAIL** — `sing-box` ran but the fetch failed/timed out ⇒ the tunnel is provably broken.
- **WARN** — `sing-box` binary absent, or the probe couldn't be set up (no UUID in the config,
  key derivation failed, ephemeral-port bind failed) ⇒ inconclusive, do not false-FAIL.

This is the heaviest, most fragile check; it degrades to WARN rather than a false FAIL, and always
cleans up its temp config + sing-box process.

### Deep probe 2 — cloudflared tunnel health (EU, WARN-only)

`cfInfo(cfg.Cloudflare.TunnelID)` → run `cloudflared tunnel info <id>` and look for live
connectors. **PASS** if ≥1 connector; **WARN** if cloudflared is absent, the output can't be
parsed, or zero connectors. Never hard-FAIL (the parse is fragile and `cloudflared` may be
managed out-of-band). Skipped entirely when CF isn't configured (`cfg.Cloudflare.TunnelID == ""`).

### Deep probe 3 — TLS cert expiry (folded into the EU sub-endpoint probe)

The sub-endpoint `httpProbe` returns the served peer certificate's `NotAfter`. doctor emits a
**WARN** when it is < 30 days away (a cert nearing expiry on a non-CF-managed front). REALITY
keypairs and the pull token **do not expire**, so there is nothing else to check — documented as
such, no separate check.

## `keys.PublicFromPrivate`

Add `func PublicFromPrivate(privB64 string) (string, error)` to `internal/keys`: base64-raw-url
decode the 32-byte X25519 private scalar, compute `curve25519.X25519(priv, curve25519.Basepoint)`,
return the base64-raw-url public key. Used by deep probe 1 to reconstruct the client public key
the RU node doesn't otherwise store. (Mirrors the encoding `keys.NewX25519Keypair` already uses.)

## Output / exit

```
RDDA doctor — RU (entry)
  ✓ units                 rdda-singbox active
  ✓ REALITY dest          addons.mozilla.org:443 reachable (TLS 1.3)
  ✓ control channel       https://sub.example.com/ru/config → 200
  ✓ last pull             2m ago
  ✓ end-to-end egress     fetched https://www.cloudflare.com/cdn-cgi/trace via the tunnel
0 warnings, 0 failures
```
Non-PASS lines append a `→ hint`. Exit 0 unless ≥1 FAIL (then exit 1 with a trailing
`N failure(s)` line).

## Error handling

- A probe's own error (timeout, refused, unparseable) becomes the check's FAIL/WARN detail — it
  never panics or aborts the run; remaining checks still execute.
- Missing inputs degrade gracefully: no `pull.env` ⇒ the control-channel check WARNs ("pull not
  configured"); no `singbox.json` on RU ⇒ the dest/egress checks WARN.
- The egress probe always removes its temp dir and kills its sing-box (even on timeout).

## Testing

**Unit:**
- `Render` + `AnyFail`: a fixture `[]Check` mixing PASS/WARN/FAIL renders the right glyphs/hints
  and `AnyFail` is true iff a FAIL is present; a WARN-only set does NOT fail.
- Role detection: `config.yaml` present ⇒ EU check list; absent ⇒ RU list (via faked seams so no
  network/sing-box).
- Each check via faked seams: unit-active map → unit check; httpProbe returning 200 / 500 /
  near-expiry cert → control/sub checks + the expiry WARN; dialDest error → dest FAIL; egress
  seam returning ok/!ok/setupErr → PASS/FAIL/WARN; cfInfo connectors/err → PASS/WARN.
- `keys.PublicFromPrivate`: derive the public key from a known private key and assert it equals
  the public key `NewX25519Keypair` produces for that pair (generate a pair, derive from its
  private, compare).

**Integration (multi-host harness):** after provisioning, run `rdda doctor` on RU with
`--probe-url http://target.rdda.test/` and assert exit 0 — exercising the core checks **and** the
real end-to-end egress probe (throwaway sing-box → live RU→edge→EU→target). No new provisioning;
doctor reads the state the harness already lays down.

## Out of scope (later / other WS)

Auto-remediation (that's W5 `heal`), a `--json` machine-readable mode, continuous/scheduled
doctor runs, and lifting the shared dial/extract helpers into a common package.
